# WINVOICE — 音声馬券メーカー

音声で馬券の買い目を作成し、netkeiba（中央）・楽天競馬（地方）へ自動入力するStreamlitアプリです。
netkeibaの馬メモを使った独自ランキング機能「**Myロジックで分析**」を搭載しています。

## 主な機能

- 🎙️ 音声入力による買い目作成（「東京8レース 単勝6番500円」など）
- 🐴 馬名だけでの買い目指定（当日出走馬から自動検索）
- 🤖 AI自然言語解釈（「3連単が100倍つく買い目を全部」など・要Anthropic APIキー）
- 📊 netkeiba厳選予想の購入・印集計・買い目提案
- 🏇 netkeiba（IPAT連携）・楽天競馬への買い目自動入力
- 🧮 **Myロジックで分析** — netkeibaの馬メモに記録した独自タイムで出走馬をランキング

## 動作環境

- Python 3.10 以上（開発確認は 3.14）
- macOS / Linux（Playwright が動作する環境）

## セットアップ

```bash
# 1. 依存関係のインストール
pip install -r requirements.txt

# 2. Playwright ブラウザのインストール（初回のみ）
python -m playwright install chromium

# 3. 環境変数ファイルの作成
cp .env.example .env
# → .env を編集して必要な値を入力（下記参照）

# 4. 起動
streamlit run voice_bet_app.py
```

http://localhost:8501/ で開きます。

## 認証情報の設定（重要）

**ユーザーが実際にIDやパスワードを入力するファイルは、プロジェクトルートの `.env` です。**
（`cp .env.example .env` で作成。`.env` は `.gitignore` 済みでGitへは含まれません）

```text
# .env の記載例（値は自分のものに置き換える）
NETKEIBA_LOGIN_ID=your-mail@example.com
NETKEIBA_PASSWORD=（netkeibaのパスワード）
APP_PASSWORD=（アプリを開くための任意のパスワード）
```

読み込みの優先順位は次のとおりです。

1. `.streamlit/secrets.toml`（Streamlit Secrets — クラウド公開時はこちら）
2. OSの環境変数
3. プロジェクトルートの `.env`

Streamlit Community Cloud等へ公開する場合は、`.env` の代わりに管理画面のSecretsに同名のキーを設定してください。

### netkeibaログインの仕組み（Myロジック分析）

1. **通常は設定不要**：WINVOICE本体のサイドバー「🔐 netkeiba ログイン設定」でログインすると、
   そのブラウザセッション（`~/.netkeiba_ipat_profile`）を自動で利用します。
2. セッション切れ時は `.env` の `NETKEIBA_LOGIN_ID` / `NETKEIBA_PASSWORD` で自動再ログインします。
3. 取得したCookieは `~/.winvoice/nk_cookies.json`（リポジトリ外・権限600）にのみ保存されます。

### アプリ簡易認証（APP_PASSWORD）

- `APP_PASSWORD` を設定すると、起動時にパスワード入力ゲートが表示されます。
- **ローカルのみで使う場合は未設定でも動きます**（画面は通常どおり表示）。
- **スマートフォン等から外部アクセスできる形で公開する場合は必ず設定してください。**
- サイドバーの「🔒 ログアウト」で認証を解除できます。

## Myロジックで分析

netkeibaの馬メモ（レース別メモ）に `1:35.5-22.4、-24` 形式で記録した独自タイムを使い、
出走馬をランキングします。

### 使い方

1. 音声または手動でレースを認識 →「🧮 Myロジックで分析」ボタン
2. または画面下部の「Myロジックで分析」セクションで race_id（12桁）を直接入力 → 過去レースも分析可
3. 分析結果は同じ画面に表示。結果はSQLiteに履歴保存され、「🗂 分析履歴」から再表示できます
4. 「📋 結果をコピー」からタブ区切りでコピーでき、スプレッドシートへ貼り付けられます
5. 「🔄 最新情報で再分析」でキャッシュを使わず再取得します

### 厩舎の話（競馬ブック連携）

分析時に競馬ブック（s.keibabook.co.jp）の「厩舎の話」を自動取得し、
各馬の詳細カードに表示します。

- race_idはnetkeiba→keibabook形式へ自動変換（場と回の入れ替え＋場コード変換）
- **全頭分の取得にはkeibabookログインが必要**（未ログインは先頭数頭のみ）
- 認証情報は `.env` の `KEIBABOOK_LOGIN_ID` / `KEIBABOOK_PASSWORD`、または
  旧プロジェクトの `~/Documents/claude-work/keibabook_config.json` を自動利用
- セッションCookieは `~/.winvoice/kb_cookies.json`（リポジトリ外）に14日キャッシュ
- 取得失敗しても分析自体は継続します（警告表示のみ）

### 地方競馬（NAR）対応

地方レースのrace_id（例: `202654071210`）を指定すると、馬メモの代わりに
**出走馬DBページの戦績表**から採用タイムを自動合成して同じルールで分析します。

- 採用タイム = 走破タイム − 上り3F（例: 1:46.4 − 38.8 → `1:07.6-38.8、-19`。末尾は馬場指数）
- 距離補正の分母は600m（上り3ハロン基準）: `上り × 距離差 ÷ 600`
- TargetHorse判定は戦績表の「着差」列（勝ち馬との秒差）を使用
- 候補範囲: 直近10走 → 有効候補が無ければ直近1年以内まで拡大
- 馬場指数の取得にはnetkeibaプレミアムログインが必要
- 厩舎の話（競馬ブック）は中央のみ

### ランキングロジック概要

- 馬メモから `先頭タイム-ラスト400m、独自数値` 形式の最初のタイムを抽出
- 今回と同距離の最速タイムを基本候補に、距離差200m以内の別距離とも比較
- 過去レースが短い場合は `ラスト400m × 距離差 ÷ 400` 秒を加算補正（長い場合は補正なし）
- 距離差400m以上しかない馬は「記録なし」
- 候補レースの公式結果でTargetHorse（勝ち馬。自身が勝ち馬なら2着馬）との
  走破タイム差が **0.6秒以上なら除外** し、次点候補へ
- 採用タイムの速い順にランキング（同タイムは馬番順、記録なしは最下部）

## データの保存先

| 内容 | 場所 |
|---|---|
| 分析履歴・キャッシュ（SQLite） | `data/winvoice_mylogic.db` |
| 分析ログ | `data/mylogic.log` |
| netkeibaセッションCookie | `~/.winvoice/nk_cookies.json`（リポジトリ外） |
| netkeibaログイン済みブラウザ | `~/.netkeiba_ipat_profile`（リポジトリ外） |

`data/` はまるごと `.gitignore` 済みです。キャッシュはバージョン付きのため、
解析ロジック更新後も古いキャッシュが悪影響を与えません。

## テストの実行

```bash
python -m pytest tests/ -v
```

タイム解析・秒変換・距離補正・候補選定・TargetHorse判定・HTMLパーサー・
SQLite保存の72テストが含まれます（通信なしで実行可能）。

## Gitへ含めてはいけないファイル

以下は `.gitignore` に登録済みです。**手動でも絶対にコミットしないでください。**

- `.env` / `.env.local`（netkeibaパスワード等）
- `.streamlit/secrets.toml`
- `data/`・`*.db`（分析履歴DB）
- `~/.winvoice/nk_cookies.json`（認証Cookie — そもそもリポジトリ外）
- `voice_bet_app_backup_*.py`（バックアップ）

## スマートフォンから使う（外部公開）の注意

このアプリはローカルPC前提の機能（Playwrightによる可視ブラウザでのIPAT自動入力・
AppleScript連携・マイク入力）を含むため、**クラウド公開との相性に制約があります。**

| 方式 | 評価 | 備考 |
|---|---|---|
| **自宅PCで起動 + Tailscale** | ◎ 推奨 | 全機能そのまま・追加費用なし・外部非公開。スマホにTailscaleを入れ `http://PC名:8501` へ |
| Cloudflare Tunnel | ○ | 全機能可・要APP_PASSWORD（URLを知っていれば誰でも到達可能） |
| Streamlit Community Cloud | △ | Playwrightの可視ブラウザ不可＝**買い目自動入力・IPAT連携は使えない**。SQLiteも再デプロイで消える。Myロジック分析のみなら動作見込み（netkeibaのクラウドIP制限リスクあり） |
| Render / Railway / Fly.io | △ | 同上＋Playwright headlessは動くが可視ブラウザ機能は不可。永続ディスクの追加設定が必要 |

**推奨は「自宅PCで起動し続け、Tailscale経由でスマホから開く」**です。
公開する場合の共通注意：

- `APP_PASSWORD` を必ず設定する
- Secretsはホスティング側のSecrets機構で管理する（`.env` をアップロードしない）
- netkeibaへのアクセス元がクラウドIPになると、ログインやスクレイピングが
  制限される可能性があります（このアプリはアクセス間隔・リトライ上限・
  403/429検知を実装済みですが、保証はできません）
- netkeibaの利用規約とアクセス負荷に配慮し、個人利用の範囲で使用してください

## 既知の制約・リスク

- netkeibaのHTML構造変更で取得が壊れる可能性があります。障害時は
  `data/mylogic.log` に解析段階・URL・ステータスが残ります
- 馬メモのタイム形式は `1:35.5-22.4、-24` / `1:09.0-22.4` 形式のみ対象です
  （コロンなしの秒表記のみのメモは対象外）
- TargetHorse判定は `db.netkeiba.com` の公式結果を利用します。結果が
  取得できないレース（海外・一部地方など）は「判定不能」として採用は有効のまま
  警告表示します
